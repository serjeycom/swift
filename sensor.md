
## 🏠 **Задание: Умный мониторинг дома**

Создайте приложение для мониторинга умного дома, которое будет отслеживать датчики

### 🎯 **Требования к реализации**

#### 1. **Модель данных**

Создайте структуру `Sensor`, которая представляет собой умный датчик, должна содержаться информация о:

 - **Комнате**, в которой установлен датчик 
 - **Название** датчика
 - **Состояние датчика** *(Села батарейка, Выключен, Включен, Сработал)* 
 -  **Заряд батареи**
 - **Текущее состояние датчика**:
 -  -  Возвращает информацию, о комнате и состоянии датчика *("Комната \(такая-то) датчик в состоянии: \(таком-то)")*
 - **Является ли данный сенсор датчиком протечки** ?
 -  - Является, если в названии содержится слово "water" иначе – нет
 - **Обновить значение заряда батареи**. Учесть:
 -  - Так как заряд батареи меняется крайне редко – пользователь не может посчитать его больше 1го раза за время работы приложения (пока не закроет)
 - **Провести диагностика датчика**. Учесть:
 -  -  Диагностика – сложный процесс, требует долгих рассчётов, пользователь крайне редко запрашивает диагностику.


#### 2. Функции

 - Добавить функцию, которая **принимает параметры Заголовок и описание пуша**. 
Отправляет Пуш-уведомление, в котором содержится **информация о сработавшем датчике + Заголовок и описание** (Достаточно вывести эту информацию в `print`)

#### 3. Создать набор разных датчиков

 - Из всех датчиков создать новый список, в котором присутвуют только датчики протечки
 - Добавить функцию, которая принимает массив датчиков и проверяет, что заряд батареи больше 20%, если заряд меньше – такой датчик необходимо дабавить в список, затем вывести в `print` число датчиков с малым зарядом и информацию о них

#### 4. **Массив для хранения датчиков**

 - Для каждого из датчиков, заряд которых меньше 20% – в название нужно добавить предупреждение "ОПАСНО ! НИЗКИЙ ЗАРЯД". Вывести в принт, убедитсья что получилось.
