
# Задания по теме "Классы и инициализаторы" на примере кофейного приложения

Представьте, что мы разрабатываем приложение для кофейни `SergioCoffee`. 

## Уровень 1: Базовые задания

### Задание 1.1: Класс `Coffee` 

Создайте класс `Coffee` со следующими характеристиками:
- Свойства: 
  - `name`  - название кофе (например, "Эспрессо", "Латте")
  - `price`  - цена в рублях
  - `volume`  - объем в миллилитрах
- Инициализатор, принимающий все три параметра
- Метод `getDescription()`, возвращающий строку вида: "Название: объем мл, цена руб."


### Задание 1.2: Класс `CoffeeMachine` с вычисляемыми свойствами

Создайте класс `CoffeeMachine`:
- Свойства: 
  - `waterLevel`  - уровень воды в литрах
  - `beansLevel`  - уровень зерен в килограммах
  - `milkLevel`  - уровень молока в литрах
- Инициализатор с начальными значениями всех уровней
- Вычисляемое свойство `isReadyToMakeCoffee: Bool`, возвращающее `true`, если **воды больше 0.1 л** и **зерен больше 0.05 кг**
- Вычисляемое свойство `canMakeCappuccino: Bool`, возвращающее `true`, если есть молоко и `кофемашина готова`
- Метод `makeCoffee()`, который уменьшает `waterLevel` на `0.2`, `beansLevel` на `0.02` и выводит сообщение о том, что кофе готов **или сообщение о том, почему кофе не может быть готов**

### Задание 1.3: Несколько инициализаторов для CoffeeOrder

Создайте класс `CoffeeOrder`:
- Свойства:
  - `coffee` (Coffee) - заказанный кофе
  - `quantity` (Int) - количество
  - `customerName` (String) - имя клиента
  - `isTakeaway` (Bool) - навынос или нет
- Назначенный инициализатор со всеми параметрами
- Удобный инициализатор только с кофе и именем (`quantity = 1`, `isTakeaway = false`)
- Удобный инициализатор с кофе, именем и количеством (`isTakeaway = false`)
- Метод `totalPrice() -> Double`, возвращающий общую стоимость

## Уровень 2: Средний уровень

### Задание 2.1: Иерархия кофейных напитков

Создайте базовый класс `Beverage`:
- Свойства: 
  - `name` (String) - название напитка
  - `basePrice` (Double) - базовая цена
  - `volume` (Int) - объем
- Инициализатор с параметрами
- Метод `getDescription() -> String`, возвращающий базовое описание

Создайте подкласс `CoffeeDrink`, наследующий от `Beverage`:
- Дополнительные свойства:
  - `strength` (Int) - крепость от 1 до 5
  - `hasCaffeine` (Bool) - содержит кофеин
- Свой инициализатор со всеми параметрами (включая базовые)
- Переопределенный метод `getDescription()`, добавляющий информацию о крепости

Создайте подклассы:
- `Espresso` (дополнительное свойство `isDouble: Bool`)
- `Latte` (дополнительное свойство `milkType: String` - "обычное", "овсяное", "миндальное")
- `Cappuccino` (дополнительное свойство `cinnamon: Bool` - с корицей или нет)

Каждый подкласс должен иметь свой инициализатор и переопределять `getDescription()`.
