
# 💅 Beauty Salon Protocol

## 🎯 Концепция

Представьте, что вы — **директор сети салонов красоты**. Вы хотите открыть новый салон, но не хотите переписывать логику работы каждый рази вам нужно, что бы все салоны работали по единому стандарту

Вы просто берёте готовый **протокол** и говорите: *«Новый салон, работай по этому контракту!», теперь салон должен исполнить все обязательства протокола.

Клиенты и мастера даже не заметят разницы между салонами — для них существует **только протокол**, а не конкретный адрес. 

Если в реальной жизни я возьму Администратора Петю и мастера Машу и пошлю их с Лиговского салона под Донбасс – они заметят разницу.

Но так быть не должно – благодаря протоколу они вообще не почувствуют изменений так как они работают только с протоколом, а не с конкретным салоном.

> 🧠 *В этом и есть сила протоколов: мы пишем один раз, а реализаций может быть сколько угодно. Главное — все они следуют единому контракту.*

---

`Date` замени на простой `Int`, что бы мы могли создавать записи в таком виде: `1,2,3,4...24`а не мучиться с датой

Тогда нам будет проще реализовать функцию свободных мастеров.

import Foundation

/// Протокол, определяющий единый контракт для любого салона красоты
protocol BeautySalonProtocol {
   
    // MARK: - Требования к салону
    
    /// Список всех сотрудников
    var employees: [Worker] { get }
    
    /// Получить сотрудников по должности
    /// - Parameter position: Должность (мастер, администратор, уборщик)
    /// - Returns: Массив сотрудников указанной должности
    func getEmployees(by position: Position) -> [Worker]
    
    /// Найти свободного мастера для процедуры
    /// - Parameters:
    ///   - procedure: Название процедуры
    ///   - timeSlot: Желаемый час записи (14...22)
    /// - Returns: Свободный мастер
    func findAvailableMaster(for procedure: Тут тип данных процедуры, at timeSlot: Int) -> Employee?

	// Нужно найти мастеров, которые умеют делать такую процедуры, и которые свободны в этот час.
	// Хотя бы словами подумай как мастер должен находить свободные часы в своих записях
    
    /// Создать новую запись
    func createBooking(тут какие-то параметры для записи) -> Booking?

	// Когда мы создаём запись – у конкретного мастера, должна добавиться запись в его список, это поможет нам контролировать свободен ли мастер
    
    /// Получить список записей в салоне
    /// - Parameter status: Статус записей (nil — все значит все записи)
    /// - Returns: Отфильтрованный массив записей
    func getBookings(by status: BookingStatus?) -> [Booking]
    
    /// Выдать зарплату за день всем в зависимости от их базовой ставки, 
    /// премию только в случае если клиентов у мастера было больше 10ти
    /// - Returns: Словарь [ID сотрудника: начисленная сумма]
    func paySalaries() -> [Worker: Int] // Работник : Зарплата
}

## 📌 Вывод

Протоколы — это **контракт**. Они позволяют:

1. **Абстрагироваться от деталей** — не важно, какой именно салон, важно, что он умеет
    
2.  **Заменять реализации** — можно подставить любой объект, подписанный под протокол
    
3.  **Писать гибкий код** — функции работают с протоколом, а не с конкретным типом
    
4.  **Тестировать** — легко подменить реальный салон на мок-объект для тестов
